# chunks-log-analyzer-skill

Репозиторий для эволюционной разработки скила `chunks-log-analyzer` — оркестратора параллельного аудита лог-файлов с использованием AI-агентов.

Этот репозиторий содержит полный цикл разработки скила: от идеи и прототипа через итеративное тестирование к стабильной версии.Такой подход позволяет отслеживать эволюцию скила, понимать, какие изменения привели к улучшению, и безопасно экспериментировать с новыми подходами.

## Что делает скил

Скил анализирует лог-файлы любого размера, разбивая их на чанки и распределяя анализ между несколькими параллельными агентами. Каждый агент анализирует свой чанк, после чего оркестратор собирает результаты, выявляет тренды и аномалии между чанками, и формирует итоговый отчёт.

Когда вы запускаете скил на большом лог-файле (гигабайты), возникает проблема: модель не может обработать такой объём информации за один запрос. Скил решает эту проблему, разбивая лог на части и делегируя анализ каждой части отдельному агенту. Главная ценность — в способности увидеть картину целиком: как ошибки распределены по времени, есть ли тенденция к ухудшению или улучшению, какие проблемы возникли в одном чанке и повторились в других.

**Как это работает:**

1. **Нарезка лога** — скил автоматически делит файл на чанки заданного размера (по умолчанию ~100KB)
2. **Параллельный аудит** — каждый чанк анализируется отдельным агентом, который ищет ошибки, предупреждения, аномалии
3. **Сбор метрик** — оркестратор собирает количественные данные от всех агентов: сколько раз встретилась каждая ошибка, когда впервые и в последний раз
4. **Выявление трендов** — сравнение метрик между чанками позволяет обнаружить ухудшение (ошибок становится больше) или улучшение (меньше)
5. **Финальный отчёт** — агрегированный результат с детализацией по категориям и выводами о состоянии системы

**Ключевые возможности:**
- Нарезка больших логов на чанки заданного размера
- Параллельный аудит каждого чанка отдельным агентом
- Обязательный формат вывода с точными счётчиками и временными метками
- Сбор метрик между чанками для выявления трендов (ухудшение/улучшение)
- Проверка полноты вывода перед отправкой результата

## Использование скила

Вызовите скил через `/chunks-log-analyzer` с путём к лог-файлу:

```
/chunks-log-analyzer /path/to/app.log
/chunks-log-analyzer ./logs/production.log
```

Скил автоматически:
1. Нарезает лог на чанки (по размеру)
2. Запускает параллельный аудит каждого чанка
3. Собирает метрики и выявляет тренды между чанками
4. Формирует финальный отчёт с выводами

## Структура

```
chunks-log-analyzer.md               # текущий скил (main = стабильная версия)
CHANGELOG.md                    # история версий: дерево веток, прогоны, проблемы
.claude/commands/
  update-skill-changelog.md     # команда: аудит и актуализация CHANGELOG.md
  record-run.md                 # команда: фиксация прогона + проблемы
```

## Рабочий процесс

1. `main` содержит последнюю стабильную версию скила
2. Каждая итерация — отдельная ветка: `git checkout -b v2-имя-фикса`
3. На ветке редактируешь `chunks-log-analyzer.md`, тестируешь, фиксируешь в `CHANGELOG.md`
4. Когда стабильно — мёрджишь в `main`

### Команды

**`/update-skill-changelog`** — аудит и приведение в порядок CHANGELOG.md:
- Проверяет структуру всех веток (статусы, родители, ссылки)
- Перестраивает Mermaid-диаграмму и оглавление
- Анонимизирует все данные (проекты → `Проект A`, `Проект B`, ...)
- Создаёт новые версии, фиксирует слияния

```
/update-skill-changelog                              # полный аудит
/update-skill-changelog new-version v2-fix-read      # новая ветка
/update-skill-changelog merge стабильно              # слияние в main
```

**`/record-run`** — фиксация прогона скила:
- Добавляет строку в таблицу прогонов (модель, проект, результат)
- Добавляет проблемы с привязкой к прогону и модели
- Автоматически анонимизирует проекты

```
/record-run sonnet, проект X, 5 чанков, УСПЕХ, 45 команд
/record-run opus, проект Y, ПРОВАЛ. Проблемы: агент читал лог напрямую
```
